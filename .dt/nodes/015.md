# 前端布局算法设计

## 已知前提
- 前端用 ReactFlow 渲染节点和边
- 数据模型：节点通过 `edges(type=to)` 表达父子关系，`root:true` 标记根节点
- 需求：初次加载自动布局、内容更新不重置位置、支持用户手动拖拽、支持一键整理

## 认知起点
前端布局需要解决两个相互冲突的需求：
1. **自动布局**：节点多时不能手动摆放，需要算法自动计算层级位置
2. **位置保留**：用户拖拽调整后，内容更新（WS 推送）不应重置位置

## 推导逻辑

### 一、布局算法选型：Dagre

**Dagre**（Directed Acyclic Graph Rendering）是专为有向无环图设计的布局库。

核心参数：
```
rankdir: 'TB'    → 从上到下（Top to Bottom）
ranksep: 80px    → 层与层之间的垂直间距
nodesep: 48px    → 同层节点之间的水平间距
marginx/y: 64px  → 画布边距
```

算法流程：
1. 把所有节点按"rank（层级）"分组——根节点 rank=0，每深一层 +1
2. 同层节点水平排列，Dagre 内部用启发式算法减少边的交叉
3. 输出每个节点的 `(x, y)` 中心坐标，减去节点尺寸的一半得到左上角坐标

当前节点树的预期布局：
```
                002
        ┌───────┼────────┐──────┐
       003     004      005    010
                     ┌──┼──┐  ┌──┴──┐
                    006 007 008 011  012
                    009 013 014
                         |
                        015
```

**为什么不用其他算法：**
- `力导向布局（Force-directed）`：节点位置不稳定，每次结果不同，不适合决策树
- `手动分层`：需要自己实现 rank 计算和交叉最小化，工作量大
- `ELK（Eclipse Layout Kernel）`：功能更强但体积大，对当前规模过重

### 二、位置缓存策略

**问题根源：** ReactFlow 在 `setNodes` 后会立即触发内部 `position change` 事件（初始化阶段节点在 `x:0, y:0`），如果监听这些事件记录位置，会把正确的 Dagre 坐标覆盖掉。

**解决方案：两层位置缓存**

```
layoutPositions（ref）
  └── Dagre 计算的位置
  └── 首次加载 或 有新节点时更新
  └── 新节点单独计算位置，已有节点不覆盖

userPositions（ref）
  └── 用户拖拽后的位置
  └── 只在 onNodeDragStop 时写入（drag 结束事件，不是 position change）
  └── 优先级高于 layoutPositions
```

**位置读取优先级：**
```
userPositions.get(id)        → 用户拖过这个节点
  ?? layoutPositions.get(id) → Dagre 计算的初始位置
  ?? { x: 0, y: 0 }          → 兜底（理论上不会到这里）
```

**触发条件：**
| 事件 | 是否重算布局 | 说明 |
|------|------------|------|
| 首次加载 | ✅ | 全部节点重算 |
| 新增节点 | ✅ | 只给新节点赋位置，已有节点保留 |
| 删除节点 | ✅ | 清理缓存，重算 |
| 内容更新（status/summary） | ❌ | 只更新 data，位置不变 |
| 用户拖拽 | ❌（写入 userPositions） | drag 结束时记录 |

### 三、一键整理

用户拖乱后可以一键恢复 Dagre 自动布局：

```
App.tsx 维护 layoutTrigger（计数器）
  → 点击 ⊞ 按钮 → layoutTrigger + 1
  → DtCanvas useEffect 监听 layoutTrigger
  → 清空 userPositions
  → 重算 Dagre（全量）
  → setNodes 更新所有节点位置
```

### 四、已知局限

- **多父节点**：Dagre 是树形布局，多父节点（同一节点有两条 from 边）会被放在其中一个父节点下，另一条 from 边显示为"跨层连线"（不影响功能，视觉上可能有斜线）
- **超大图**：节点数 > 100 时 Dagre 性能会下降，目前规模不需要考虑
- **跨项目节点**：`target` 含 `::` 的边暂时不参与布局计算

## 讨论共识
- ✅ 使用 Dagre TB 方向自动分层布局
- ✅ 两层位置缓存：layoutPositions（Dagre）+ userPositions（drag）
- ✅ 用 onNodeDragStop 而非 onNodesChange 记录用户位置，避免 ReactFlow 内部事件污染
- ✅ 内容更新不重置位置，只有新增/删除节点时重算布局
- ✅ 一键整理（⊞ 按钮）清空 userPositions 并重算 Dagre
