# 前端展示——决策树可视化看板的设计

## 已知前提

- 来自 [[008]] CLI 设计共识：**前端是看板，交互在终端**
- 前端角色：将 `.dt/` 文件结构可视化，不是交互界面
- 人的认知模式分两种：总揽全局（树形图）和局部细节（节点深入）
- 底层数据结构是有向图（DAG），不是严格的树
- 现有实现：`dt serve` 启动 HTTP + WebSocket 服务，有基础的前端框架

## 认知起点

看板的核心目的：**让人一眼看清决策树的全局状态，快速定位关注的节点**。

需要回答的问题：
1. 用什么方式展示树/图结构？
2. 节点展示哪些信息，点击后展示什么？
3. 跨分支 related 边如何显示？
4. 状态（pending/in_progress/decided/completed/rejected）如何视觉区分？
5. 是否需要实时更新（文件变化时自动刷新）？

## 推导逻辑

### 展示模型

Canvas 画布 + 卡片节点是正确的形态：
- 节点以卡片形式浮在画布上，支持拖拽布局
- 边是带标签的有向线段，标签显示 edge.summary
- 跨分支 related 边与父子 from/to 边视觉上需要区分（待视觉设计）

### 节点信息密度（渐进式披露）

```
默认：title + type + status（卡片缩略形态）
点击展开：+ summary
正文区：Markdown 渲染，支持切换为编辑模式
```

### 编辑与保存

- frontmatter 字段（title/summary/status/type）：通过 `dt update` API 更新
- 正文内容：直接写回 `.dt/nodes/xxx.md` 文件
- 保存触发：显式保存按钮（或 Ctrl+S），不自动保存防止误操作

### 重写策略

完全重写 `src/public/`，不复用现有 app.js（现有实现是基础骨架，不满足新需求）。
后端 `src/server/` 保留，API 层按需扩展。

## 讨论共识

- ✅ Canvas 画布 + 卡片节点
- ✅ 边带标签（显示 edge.summary），方向箭头
- ✅ 节点信息渐进式披露：默认缩略 → 展开 summary → 正文 Markdown 渲染
- ✅ 正文支持 Markdown 渲染 + 切换编辑模式，编辑后显式保存
- ✅ 完全重写前端，后端保留
- ✅ 视觉设计参考 **Nothing Design System**（瑞士排版、工业风、单色调）

### Nothing Design System 关键约束

**调色板（暗色模式）**
- 背景 `#000000`（OLED 黑），卡片 `#111111`，边框 `#333333`
- 文字层级：display `#FFFFFF` → primary `#E8E8E8` → secondary `#999999` → disabled `#666666`
- 状态色：completed `#4A9E5C`，pending `#D4A843`，rejected/error `#D71921`

**字体**
- UI 文本：Space Grotesk（body/title）
- 标签/数据：Space Mono，ALL CAPS，letter-spacing 0.06em
- 显示级别：Doto（仅用于 hero 标题，如项目名）

**组件规则**
- 无阴影、无渐变、无 border-radius > 16px
- 状态反馈用行内文本 `[SAVED]` / `[ERROR: ...]`，不用 toast
- 边：细线 1–1.5px；父子边实线，related 边虚线
- 节点卡片：12–16px radius，1px `#333333` border，16–24px padding

**已确定**
- ✅ Canvas 渲染库：**React Flow**
- ✅ 前端框架：**React + TypeScript**
- ✅ 构建工具：**Vite**（输出到 `src/public/`，由现有 Express 服务器托管）

- ✅ 模式：**亮色优先**（暖白 `#F5F5F5` 底，黑墨印刷手册风格）
