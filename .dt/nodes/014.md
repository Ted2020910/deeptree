# 多项目管理——一个 serve 如何支持多个分散的 dt 项目

## 已知前提

- 现有架构：单项目模式，`dt serve` 读当前目录的 `.dt/`，前端只展示一棵树
- 项目文件分散在各自代码目录，每个项目有独立 `.dt/` 文件夹
- `dt init` 已支持在项目目录初始化 `.dt/` 结构

## 认知起点

核心矛盾：**项目文件是分散的，但 serve 入口是统一的**。
需要解决两件事：
1. 后端如何知道世界上有哪些 dt 项目（发现机制）
2. 前端如何让用户在项目间切换（交互机制）

## 推导逻辑

### 后端：全局注册表

**决策：** 使用 `~/.dt/projects.json` 作为全局注册表。

```json
{
  "projects": [
    { "id": "dt",  "name": "dt",  "path": "/Users/xxx/code/dt",  "addedAt": "..." },
    { "id": "app", "name": "app", "path": "/Users/xxx/code/app", "addedAt": "..." }
  ]
}
```

**注册时机：**
- `dt init <name>` 时自动写入注册表（新项目自动可见）
- 新增 `dt register` / `dt unregister` 命令手动管理已有项目

**serve 启动：**
- 读取注册表，校验每个 path 下 `.dt/` 是否存在
- 路径不存在的项目标记为 `unreachable`，前端灰显但不报错

**API 变更（保持后向兼容）：**

```
旧（保留兼容）:
  GET /api/tree  →  返回 serve 启动目录的项目（无注册表时的降级）

新:
  GET  /api/projects                         → 项目列表
  GET  /api/projects/:id/tree                → 某项目完整树数据
  PUT  /api/projects/:id/nodes/:nid/frontmatter
  PUT  /api/projects/:id/nodes/:nid/content
  WS   /  →  推送 { projectId, type: 'change' }
```

**文件监听：** 对所有注册项目的 `.dt/nodes/` 目录同时监听，变更时 WS 推送带 projectId。

### 前端：Header 项目选择器

**布局（在现有 Header 基础上）：**

```
┌─────────────────────────────────────────────────────────┐
│  dt·  [当前项目名 ▾]      ████░░░   8 NODES  [LT] │
└─────────────────────────────────────────────────────────┘
         ↑ 点击展开项目列表
```

项目选择器（Space Mono 样式的下拉）：
```
┌─────────────────┐
│ ● dt-cli        │  ← 当前，绿点
│   app-backend   │
│   biz-model     │
│ ✕ lost-project  │  ← unreachable，灰色
└─────────────────┘
```

**切换行为：**
- 切换后 selectedId 重置为 null（Detail Panel 关闭）
- 重新请求新项目的 `/api/projects/:id/tree`
- URL 更新为 `?project=:id`（支持刷新后保持当前项目）

**状态管理变化：**
- `useTree` hook 接收 `projectId` 参数
- App 层维护 `currentProjectId` state
- 切换时只需更新 `currentProjectId`，`useTree` 自动重新 fetch

### dt init 变更

`dt init <name>` 完成后额外执行：
```typescript
registerProject({ id: slugify(name), name, path: process.cwd() })
```
写入 `~/.dt/projects.json`，保证新项目立即在 serve 中可见。

## 讨论共识

**架构决策（已确认）：**

| 层 | 方案 | 理由 |
|----|------|------|
| 存储 | `~/.dt/projects.json` 全局注册表 | 跟随用户，`dt init` 自动注册，长期项目友好 |
| 后端 API | `/api/projects/:id/*` 路由 + 兼容旧 `/api/tree` | 不破坏现有用法 |
| 前端 | Header 中间项目选择器下拉 | 不破坏现有布局，切换成本最低 |
| 切换行为 | selectedId 重置，Detail Panel 关闭 | 简洁，无跨项目状态残留 |
| URL | `?project=:id` query param | 支持刷新保持项目，可分享链接 |

**待实现的新命令：**
- `dt register [path]` — 注册已有项目（默认当前目录）
- `dt unregister <id>` — 从注册表移除
- `dt projects` — 列出所有注册项目及状态
