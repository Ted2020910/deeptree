# dt项目改造与实现

## 已知前提（讨论共识汇总）

### 来自 [[009]] 数据模型
- 节点 = frontmatter（id/标题/摘要/类型/状态/edges）+ 自由 Markdown 正文
- 新增"摘要"字段
- 类型支持预设 + 自定义
- 边存储在节点 frontmatter 中（方向 + 摘要 + 可选详细逻辑链接）
- 支持跨分支关联，底层是有向图

### 来自 [[008]] CLI 命令设计
- 精简为 6 条命令：tree / status / show / add / link / update
- 砍掉：dt ask / dt listen / dt diff / dt snapshot / dt check
- frontmatter 走 CLI，正文 Agent 直接编辑
- Git 操作对 Agent 透明，CLI 写操作后自动 git add + commit
- 变更检查嵌入 dt tree / dt status（unstaged changes = 人类修改）
- 前端是看板，交互在终端

## 推导逻辑

### 现有代码 vs 目标设计的差异

| 维度 | 现有实现 | 目标设计 | 改动 |
|---|---|---|---|
| **节点 frontmatter** | id/type/parent/children/status/created | 新增 summary、edges | 需改造 |
| **边** | 隐含在 parent/children 中，无独立信息 | edges 数组，含 target/direction/summary | **新增** |
| **节点类型** | 固定 6 种 | 预设 + 自定义 | 需改造 |
| **CLI 命令** | 16 条 | 6 条 | **大幅精简** |
| **dt ask** | 有（阻塞轮询） | 砍掉 | 删除 |
| **dt listen** | 有（chokidar 监听） | 砍掉 | 删除 |
| **dt check** | 有（变更检查） | 嵌入 dt tree/status | 改造 |
| **dt serve** | 有（HTTP + WebSocket） | 保留（前端看板） | 暂不改 |
| **git 集成** | 无 | CLI 写操作自动 commit | **新增** |
| **dt link** | 无 | 添加跨分支关联边 | **新增** |
| **context 系统** | 有（独立 context 文件） | 待讨论是否保留 | 待定 |
| **changes 系统** | 有（变更记录文件） | 被 git diff 取代 | 可删除 |
| **interactions 系统** | 有（交互文件） | dt ask 砍掉后不需要 | 可删除 |

### 改造优先级

按影响范围和依赖关系排序：

**第一步：数据模型改造（基础）**
1. 节点 frontmatter 新增 summary 字段
2. 节点 frontmatter 新增 edges 数组
3. 节点类型支持自定义
4. 更新 types/、core/node.ts、core/frontmatter.ts

**第二步：新增功能**
5. 实现 dt link 命令（跨分支关联）
6. 实现 git 自动 commit（每次写操作后）
7. 变更检查嵌入 dt tree / dt status（git diff 检测 unstaged changes）

**第三步：精简砍除**
8. 砍掉 dt ask（删除 commands/ask.ts、core/interaction.ts）
9. 砍掉 dt listen（删除 commands/listen.ts、utils/watcher.ts）
10. 砍掉 dt check（功能迁入 dt tree/status）
11. 清理 interactions/、changes/ 相关代码

**第四步：完善**
12. 更新 dt show 输出格式（展示 edges 信息）
13. 更新 dt tree 输出格式（展示摘要、变更标记）
14. 更新 CLAUDE.md 协议（匹配新命令集）
