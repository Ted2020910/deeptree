# 人类在使用AI工具时的工作方式

## 已知前提
- 人类的认知带宽有限，无法并行处理大量信息
- 人类的优势在于判断力、价值判断、目标定义

## 推导逻辑

### 人类当前如何与 AI 协作

**典型工作流**
1. 心里有个想法（模糊或清晰）
2. 用自然语言描述给 AI
3. AI 产出结果
4. 人审查结果 → 满意则接受，不满意则重新描述
5. 循环往复

**人类的强项**
| 能力 | 说明 |
|---|---|
| 目标定义 | 知道"要什么"，即使表达不清楚 |
| 价值判断 | 能判断方案好不好、合不合适 |
| 全局感知 | 能感知项目整体方向是否对头 |
| 创意与直觉 | 能提出 AI 想不到的角度 |
| 经验常识 | 知道"现实中行不行得通" |

**人类的局限**
| 局限 | 说明 |
|---|---|
| 表达能力 | 心里清楚但说不准确，尤其是复杂需求 |
| 认知带宽 | 无法审查几万行代码，无法同时跟踪多条推导路径 |
| 注意力 | 容易被大量输出淹没，丧失判断力 |
| 一致性 | 不同时间对同一问题可能给出不同描述 |
| 持久性 | 前几轮讨论的细节可能记不清 |

### 人在协作中真正需要的

1. **低认知负荷的概览** — 快速知道"现在在哪、为什么在这"
2. **按需深入** — 想看细节时能看到，不想看时不被打扰
3. **关键时刻的决策权** — 在重要分支上有选择权，而不是事后才发现跑偏
4. **可回溯的记录** — 之前讨论了什么、达成了什么共识，随时可查
5. **跨会话的连续性** — 关掉窗口再打开，不需要从头解释

### 对 dt 设计的启示
- 前端界面是人的主要入口，必须**低认知负荷**
- 人通过**可视化决策树 + 渐进式披露**获取信息
- 人通过**提交变更**而非实时编辑来参与（从容操作，有意义的提交）
- dt 的本地文件让人可以用 Obsidian、编辑器等熟悉的工具直接查看和编辑
