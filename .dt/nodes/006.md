# AI Agent的能力边界与运作

## 已知前提
- dt 既要给人用，又要给 AI 用
- 要设计工具的形态，必须先理解使用者（AI 和人）各自怎么工作

## 推导逻辑

### AI Agent 当前是怎么运作的

以 Claude Code 为例（其他 Agent 类似）：

**输入层**
- 接收自然语言指令（用户 prompt）
- 读取系统指令（CLAUDE.md、settings）
- 读取工具定义（可用的 tools 列表及其描述）
- 读取对话上下文（之前的对话历史）

**推理层**
- 在单次推理中完成：理解意图 → 规划步骤 → 选择工具 → 生成参数
- 推理基于上下文窗口内的所有信息，窗口外的信息"不存在"
- 没有持久记忆，每次会话从零开始（除非有外部文件）

**执行层**
- 通过工具调用与外部世界交互：Bash、文件读写、搜索等
- 可以并行调度多个工具
- 执行速度极快，几秒内可产出大量代码/文档
- 执行结果作为新的上下文反馈到推理层

**AI 的强项**
| 能力 | 说明 |
|---|---|
| 速度 | 几秒内生成大量代码和分析 |
| 并行 | 可以同时探索多条路径 |
| 一致性 | 在给定前提下，推导逻辑稳定 |
| 工具使用 | 能调用 Bash、读写文件、搜索等 |
| 模式识别 | 快速识别代码模式、架构模式 |

**AI 的局限**
| 局限 | 说明 |
|---|---|
| 无持久记忆 | 会话结束，一切归零；跨会话依赖外部文件 |
| 上下文窗口有限 | 超出窗口的信息等于不存在 |
| 不知道人类真正想要什么 | 只能基于人给的描述推断，描述不准就跑偏 |
| 无法自主判断"该不该继续" | 倾向于执行到底，缺少"停下来问一问"的直觉 |
| 不理解人的认知负荷 | 输出大量信息不考虑人能不能消化 |
| 回溯缺乏外部支撑 | 能回溯，但决策脉络没有外化记录，回溯质量不可控 |

### 对 dt 设计的启示
- AI 通过 **Bash 命令** 与 dt 交互，这是最通用的接口
- AI 的上下文窗口有限，dt 命令的输出应该**简洁**，深层信息通过文件链接
- AI 没有持久记忆，dt 的本地文件承担了**跨会话记忆**的角色
- AI 不知道该不该停下来，dt 的交互模式（key_points/every_step）提供了**外部刹车机制**
